# rt-npm-test

A standalone Vite + React app that smoke-tests the **published** `@chibi3d/runtime` npm package (not the workspace source). It renders `.chibi.json` scenes with `<ChibiScene>`, exercises the imperative API (`play`/`stop`/`setPaused`) and logs `onEvent` output.

```sh
pnpm install
pnpm dev
```

## Scenes

Each scene in `public/scenes/` is generated by a script in `scripts/` and targets a different limit of the editor/runtime:

| Scene | Stress target |
| --- | --- |
| `keyboard` | Per-node states + hover interactions on every key (35 states) |
| `orrery` | Hierarchy depth — moons of moons, a 12-level nested comet tail, dozens of rotation tracks in one clip |
| `metropolis` | Node count (~900 nodes), fog, spot lights, step-keyframe (`visible`) beacon blinks |
| `gallery` | Every geometry kind at extreme params, ~74 materials, animated material color/opacity tracks |
| `arcade` | Interactions — click toggles, layered hover+click, overlapping one-shot clips, cross-node actions (knob click drives parent group) |
| `hyrule` | Lowpoly Zelda-style overworld — flat shading everywhere, one-shot clips (sword draw), pivot-group lid + paired toggles (chest), one-way collect transitions (rupees) |
| `hero` | The hand-authored demo scene from the main app |

Regenerate after schema or runtime changes:

```sh
node scripts/generate-keyboard-scene.mjs
node scripts/generate-orrery-scene.mjs
node scripts/generate-metropolis-scene.mjs
node scripts/generate-gallery-scene.mjs
node scripts/generate-arcade-scene.mjs
node scripts/generate-hyrule-scene.mjs
```

Shared builder helpers (ids, node/material/state/interaction constructors, seeded RNG, `validateDocument` + write) live in `scripts/lib.mjs`.
